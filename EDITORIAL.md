# Editorial Capabilities & Rules (AI Authoring Guide)

Use this guide when generating new posts in `src/content/blog/*.mdx`.

## Frontmatter requirements

```yaml
title: '...'
description: '...'
date: 2026-02-22
tags: ['AI', 'strategy']
category: 'AI Strategy'
sourceImage: '/images/blog/<slug>.<ext>'
coverImage: '/images/blog/og/<slug>-og.jpg'
draft: false
```

- `sourceImage` = original/local hero image source.
- `coverImage` = generated branded OG/social thumbnail.

## Charts & infographics

Do **not** hand-build charts with ad-hoc HTML/CSS bars.

Use community libraries/components:

- `src/components/BarChart.astro` (Chart.js-backed)

If you need new chart types, extend this component with Chart.js rather than custom visual hacks.

## Mermaid diagrams

Use fenced mermaid blocks:

````md
```mermaid
flowchart LR
  A --> B
```
````

Mermaid rendering is enabled in article pages via the `mermaid` npm package.

## Code snippets

Use fenced code blocks with language hints (e.g. `ts`, `js`, `bash`, `json`).

Code blocks are enhanced by `rehype-pretty-code` + copy-button transformer.

## OG/social thumbnail pipeline

Before commit/build, OG images are generated by:

- `scripts/generate_og_images.py`
- wrapper: `scripts/run_og_generation.sh`

Commands:

```bash
npm run og:generate
npm run build
```

The generator preserves `sourceImage` and updates `coverImage` to branded 1200×630 assets in `public/images/blog/og/`.
